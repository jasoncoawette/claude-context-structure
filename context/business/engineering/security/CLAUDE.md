# Security

Application and infrastructure security practices for the user's systems.

## What lives here

- `api-saftey/` — securing API endpoints (authn, authz, rate limiting, input validation)
- `enviroment-hardening/` — hardening runtime environments and configuration
- `iam/` — identity and access management, roles, and least privilege
- `package-vulnerability/` — dependency scanning and patching
- `penetration-testing/` — authorized offensive testing of systems

## How to navigate

Read the relevant leaf `SKILL.md` for task instructions; don't load the whole tree.

## Conventions

- Default to least privilege and deny-by-default.
- Only test systems you are explicitly authorized to test.
