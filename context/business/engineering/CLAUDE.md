# Engineering

Technical domain covering how the user's software is built, shipped, secured, and tested.

## What lives here

- `backend-development/` — server-side code, data stores, and APIs
- `devops/` — deployment, hosting, CI/CD, and observability
- `frontend-development/` — client-side UI work
- `security/` — application and infrastructure security practices
- `testing/` — test strategy across unit, integration, end-to-end, and QA

## How to navigate

Read the relevant leaf `SKILL.md` for task instructions; don't load the whole tree.

## Conventions

- Prefer the smallest change that solves the task; touch one domain at a time.
- Never commit secrets — use environment variables and the patterns in `security/`.
